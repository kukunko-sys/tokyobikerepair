## 検出事項と推奨事項

### 1. 画像のEXIF情報（軽微）

> [!NOTE]
> 撮影日時・撮影端末・GPS座標などのメタデータが画像ファイル（`hutari.png`, `shuri-01.png`, `shuri-02.png`）に含まれている可能性があります。

**推奨対応**: 以下のコマンドでEXIFを一括削除できます：
```bash
# ExifToolが必要（brew install exiftool）
exiftool -all= images/*.png
```

### 2. Google Fonts の外部読み込み（軽微）

> [!NOTE]
> `fonts.googleapis.com` および `fonts.gstatic.com` への外部リクエストが発生します。GDPR（EU一般データ保護規則）が厳格な地域ではIPアドレスの第三者送信が問題になるケースがありますが、**日本国内向けサービスのため実質的なリスクはありません**。

**対応不要**（日本の個人情報保護法における問題なし）

### 3. リポジトリがPublic（注意）

> [!IMPORTANT]
> GitHub Pagesを無料で使うにはリポジトリがPublicである必要があります。HTMLソースコード・画像・Gitの変更履歴がすべて公開されています。

**確認事項**:
- ✅ ソースコードに機密情報なし → 問題なし
- ✅ コミットメッセージに機密情報なし → 問題なし
- ⚠️ 今後ファイルを追加する際、**APIキー・パスワード・社内資料等を誤ってコミットしないこと**

### 4. セキュリティヘッダー未設定（情報提供）

GitHub Pagesでは以下のHTTPヘッダーをカスタム設定できません（プラットフォーム側の制約）：
- `X-Frame-Options`（クリックジャッキング防止）
- `Content-Security-Policy`（CSP）
- `Strict-Transport-Security`（HSTS）

ただし、GitHub Pages自体が以下を自動適用しているため、実害はありません：
- **強制HTTPS**（HTTPアクセスは自動リダイレクト）
- **基本的なセキュリティヘッダー**はGitHub側で管理

## 結論

本サイトは**静的HTML/CSS のみ**で構成されており、動的処理・フォーム・JavaScript が一切ないため、**Webサイトとしてのセキュリティリスクは極めて低い**です。

上記の軽微な推奨事項（画像EXIF削除、Public Repoの注意）を意識していただければ、現状のまま安全に運用可能です。
